## 4. SEO technique

### 4.1 Balises title et meta description

#### Page d'accueil

```
Title : [METIER] a [VILLE] | [NOM_CLIENT] - [SLOGAN_COURT]
Meta description : [NOM_CLIENT], votre [METIER] a [VILLE]. [PROPOSITION_VALEUR]. Decouvrez nos services et prenez rendez-vous en ligne. Tel. [TELEPHONE].
```

#### Page services

```
Title : [SERVICE_SPECIFIQUE] a [VILLE] | [NOM_CLIENT]
Meta description : [NOM_CLIENT] propose [SERVICE_SPECIFIQUE] a [VILLE]. [DESCRIPTION_COURTE_DU_SERVICE]. Demandez un devis gratuit.
```

#### Page contact

```
Title : Contact - [NOM_CLIENT] | [METIER] a [VILLE]
Meta description : Contactez [NOM_CLIENT], [METIER] a [VILLE]. Adresse : [ADRESSE]. Tel : [TELEPHONE]. Ouvert [HORAIRES_COURTS]. Formulaire en ligne.
```

#### Article de blog

```
Title : [TITRE_ARTICLE] | [NOM_CLIENT] - [METIER] [VILLE]
Meta description : [RESUME_ARTICLE_EN_155_CARACTERES]. Conseils de votre [METIER] a [VILLE].
```

> **Regles** : Title entre 50 et 60 caracteres. Meta description entre 120 et 155 caracteres. Chaque page doit avoir des balises uniques.

### 4.2 Donnees structurees Schema.org

Ajouter le balisage JSON-LD suivant sur toutes les pages du site :

```json
{
  "@context": "https://schema.org",
  "@type": "LocalBusiness",
  "@id": "[SITE_URL]",
  "name": "[NOM_CLIENT]",
  "description": "[DESCRIPTION_COURTE]",
  "url": "[SITE_URL]",
  "telephone": "[TELEPHONE]",
  "email": "[EMAIL]",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "[RUE]",
    "addressLocality": "[VILLE]",
    "postalCode": "[CODE_POSTAL]",
    "addressRegion": "[CANTON]",
    "addressCountry": "CH"
  },
  "geo": {
    "@type": "GeoCoordinates",
    "latitude": "[LATITUDE]",
    "longitude": "[LONGITUDE]"
  },
  "openingHoursSpecification": [
    {
      "@type": "OpeningHoursSpecification",
      "dayOfWeek": ["Monday", "Tuesday", "Wednesday", "Thursday", "Friday"],
      "opens": "[HEURE_OUVERTURE]",
      "closes": "[HEURE_FERMETURE]"
    }
  ],
  "image": "[SITE_URL]/images/[IMAGE_PRINCIPALE]",
  "priceRange": "[FOURCHETTE_PRIX]",
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "[NOTE_MOYENNE]",
    "reviewCount": "[NOMBRE_AVIS]"
  },
  "sameAs": [
    "[URL_FACEBOOK]",
    "[URL_INSTAGRAM]",
    "[URL_LINKEDIN]"
  ]
}
```

Pour les articles de blog, ajouter egalement :

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "[TITRE_ARTICLE]",
  "author": {
    "@type": "Organization",
    "name": "[NOM_CLIENT]"
  },
  "datePublished": "[DATE_PUBLICATION]",
  "dateModified": "[DATE_MODIFICATION]",
  "image": "[URL_IMAGE_ARTICLE]"
}
```

Pour la page FAQ :

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "[QUESTION_1]",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "[REPONSE_1]"
      }
    }
  ]
}
```

### 4.3 Optimisation des images

| Action | Detail |
|--------|--------|
| Format | Utiliser WebP ou AVIF (avec fallback JPEG) |
| Taille | Max 200 Ko par image, max 1920px de large |
| Attribut alt | Descriptif et incluant le mot-cle si pertinent. Ex: "Salon de coiffure [NOM_CLIENT] a [VILLE]" |
| Nom de fichier | Descriptif avec tirets. Ex: `coiffeur-lausanne-salon.webp` |
| Lazy loading | Activer le chargement differe pour les images sous la ligne de flottaison |
| Compression | Utiliser TinyPNG, Squoosh ou le plugin Next.js Image |

### 4.4 Bonnes pratiques techniques

- URLs courtes et lisibles : `[SITE_URL]/services/[service-en-minuscules]`
- Liens internes entre les pages (maillage interne)
- Fil d'Ariane (breadcrumb) sur toutes les pages
- Page 404 personnalisee avec liens de retour
- Temps de chargement < 3 secondes
- Pas de contenu duplique entre les pages

## 9. Notes et recommandations specifiques

### Specificites suisses

- **Multilinguisme** : Si le client est dans une zone bilingue (Bienne, Fribourg, Valais), envisager des pages dans les deux langues avec balises hreflang.
- **Domaine .ch** : Privilegier un domaine en .ch pour le referencement local suisse.
- **Annuaires suisses** : local.ch et search.ch sont les references en Suisse et apportent des backlinks de qualite.
- **Google Maps** : En Suisse, Google Maps est le premier reflexe des consommateurs pour trouver un commerce local. La fiche Google Business Profile est donc prioritaire.
- **TWINT / moyens de paiement** : Mentionner les moyens de paiement acceptes (TWINT, Postcard) car c'est un critere de recherche frequent.

### Points d'attention

- Ne jamais acheter de liens ou d'avis (penalites Google)
- Privilegier la qualite du contenu a la quantite
- Mettre a jour regulierement les horaires et informations de contact
- Repondre systematiquement a tous les avis (positifs et negatifs)
- Verifier que le site est conforme a la nLPD (nouvelle Loi sur la Protection des Donnees suisse)
